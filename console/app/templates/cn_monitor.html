The Change Notification Monitor tab proxies data from the EthosCn CNM API. To enable it, point the console at your running CNM instance.

  1. Add CNM_BASE_URL to console/.env:
    CNM_BASE_URL=http://localhost:5000 (dev)
    CNM_BASE_URL=https://your-host/prod/cnm (prod)
  2. Production only — also add a Bearer token:
    CNM_API_KEY=<Azure AD service token>
    In dev the CNM DevAuth handler accepts requests without a token.
  3. Restart the console: python run.py
  4. Return to this tab — it will load automatically when configured.
The CNM service must be running and reachable from the console host. See cnm/docker-compose.yml for local startup.
