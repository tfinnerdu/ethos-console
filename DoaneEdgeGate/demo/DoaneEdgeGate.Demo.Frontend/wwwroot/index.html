Instant Enrollment — Demo

Reproduces the actual two-stage bug that DoaneEdgeGate exists to catch. Stage 1 (browser): a date picker builds a Date at local midnight, then .toISOString() serializes it as a UTC instant a few hours into the morning for any timezone west of UTC — the date itself is still correct at this point. Stage 2 (Web API, simulated by "AddPerson demo endpoint" below): the real Colleague Web API converts that instant back to server-local (Central) time and truncates to a date — that conversion, not the browser alone, is where the calendar day actually gets lost. Watch the response's storedBirthDate field for what Stage 2 would actually store; Mode: Active on the gate fixes it by stripping to a bare date before Stage 2 ever runs. Nothing here is real — no data is stored anywhere, this just POSTs JSON to whatever address is configured below.
